ShapeFlags 的位运算原理 ​

js
const ShapeFlags = {
  element: 0,
  stateful_component: 0,
  text_children: 0,
  array_children: 0,
};
不够高效，考虑使用位运算的方式来替代
0000 0001 → element
0010 → stateful
0100 → text_children
1000 → array_children

上述为js的位运算，为什么0001表示element，不是1000
js
在 JavaScript（以及大多数编程语言）中，当使用位运算来表示标志或选项时，每个标志通常从最低有效位（最右边的位）开始分配一个唯一的位位置。
这意味着第一个标志用 `1`（二进制 `0001`）表示，第二个标志用 `2`（二进制 `0010`），第三个标志用 `4`（二进制 `0100`），以此类推。
每个标志对应于 2 的幂，确保每个标志都有一个唯一的位位置，便于通过组合这些位来唯一识别标志的组合。

以下是为什么 `0001` 表示 `element` 而不是 `1000` 的原因：

1. **位位置和 2 的幂**：
   - **最低有效位（LSB）**：二进制数中最右边的位，代表 \(2^0 = 1\)。
   - 每向左移动一位，代表下一个更高的 2 的幂：
     - 第二位：\(2^1 = 2\)（二进制 `0010`）
     - 第三位：\(2^2 = 4\)（二进制 `0100`）
     - 第四位：\(2^3 = 8\)（二进制 `1000`）

2. **标志的分配**：
   - **Element**：分配给最低有效位，表示值 `1`（二进制 `0001`）。
   - **Stateful Component**：分配给第二位，值为 `2`（二进制 `0010`）。
   - **Text Children**：分配给第三位，值为 `4`（二进制 `0100`）。
   - **Array Children**：分配给第四位，值为 `8`（二进制 `1000`）。

3. **为什么从最低有效位开始**：
   - **效率**：从最低有效位开始可以高效地利用位，并方便地使用位或（`|`）操作来组合标志。
   - **惯例**：在编程中，从最低有效位开始分配标志是一种常见的惯例，有助于可读性和一致性。

4. **组合标志**：
   - 通过使用 2 的幂，你可以在不产生冲突的情况下组合多个标志：
     - 例如，一个既是 `element` 又有 `text_children` 的标志值为 `1 | 4 = 5`（二进制 `0101`）。
     - 这个唯一的组合允许你使用位与（`&`）操作来检查特定的标志。
```
#### 运算原理
```js
| (两位都为0，才为0)    -> 修改
& (两位都为1，才为1)    -> 查找
```
