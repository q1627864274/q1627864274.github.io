如果想要生产环境也保持小图base64的格式，怎么处理？
手写一个vite插件
import fs from 'node:fs'
// 插件一个函数而不是对象，方便传值
const MyPlugin: any = (limit = 4096) => {
   return {
      name: 'my-plugin',
      // vite中使用了esbuild，rollop
      // transform属于rollop其中的一个钩子函数
       // 加载一个模块就会运行，将内容和路径给你
      async transform(code, id) {
         if (process.env.NODE_ENV !== 'development') {
            return
         }
         if (!id.ensWith('.png')) {
            return
         }
         const stat = await fs.promise.stat(id)
         if (stat.size > limit) {
            return
         }
         // 转化为base64
         const buffer = await fs.promise.readFile(id)
         const base64 = buffer.toString('base64')
         const dataurl = `data:image/png;base64,${base64}`
         return {
            code: `export default ${dataurl}`
         }
      }
   }
}
export default defineConfig({
    plugins: [
       MyPlugin()
    ])
  ],
})

el-tree组件子选项有很多的导致页面卡顿(vue2) ​

js
1. 采用组件提供的懒加载模式，点击才会进行加载字选项
2. 一次性子选项太多，点击下拉展开也会很卡顿，因为一次性dom渲染太多了，采用虚拟滚动组件和el-tree相结合，
   例如vue-easy-tree这个第三方库，内部就是vue-virtual-scroll-list与tree组件相结合
3. 实现了虚拟滚动发现，点击全部勾选，还是存在卡顿，因为内部会递归调用setCheck导致变慢
    解决方案：check-strictly设为true  父子组件不再联动    绑定check-change方法，
    内部采用this.&ref.tree.setChecked()去实现字选项的勾选，这样就减少了卡顿
